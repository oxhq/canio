# Canio v1.0.0

Canio `v1.0.0` is now publicly available on Packagist as `oxhq/canio`.

## What It Is

Canio is the Laravel package for browser-grade PDFs.

It exists for the category where classic HTML-to-PDF engines stop being enough:

- real Chromium layout
- JavaScript execution before capture
- explicit readiness with `window.__CANIO_READY__`
- render artifacts for debugging and replay
- operational rendering flows through the Stagehand runtime

## Install

```bash
composer require oxhq/canio
php artisan canio:install
```

## What Shipped

- public Packagist install path for `oxhq/canio`
- public monorepo at `oxhq/canio`
- public split distribution repo at `oxhq/canio-laravel`
- `v1.0.0` Stagehand binaries for Linux, macOS, and Windows
- checksum verification during runtime install
- embedded-mode install and first-render flow
- reproducible fidelity, fairness, and JavaScript-capability benchmarks

## Product Positioning

The claim is narrow on purpose:

`Canio is the Laravel package for browser-real, high-fidelity PDFs.`

That means:

- Canio is the right tool when fidelity, JavaScript, readiness, and operational rendering matter.
- Dompdf and mPDF can still be faster on the simplest uncached static documents.
- Canio is not trying to win the minimum-work HTML-to-PDF lane.
